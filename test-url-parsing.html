URL解析功能测试

请在Vue API Tester中测试以下URL，验证查询参数是否能正确解析到Query参数列表中：

测试用例 1: 简单查询参数

https://api.example.com/users?page=1&limit=10&sort=name
预期解析结果：
  • page = 1
  • limit = 10
  • sort = name

测试用例 2: 复杂查询参数（您提供的示例）

http://192.168.201.129:20831/code_bricks/menu?sort=-order&filter={"tenant_id":"sdfjksdjf=1212,dfjkdfjdkjf=2324,dfkdfkdjkdjf=455是的萨芬的萨芬的678"}&pk=id
预期解析结果：
  • sort = -order
  • filter = {"tenant_id":"sdfjksdjf=1212,dfjkdfjdkjf=2324,dfkdfkdjkdjf=455是的萨芬的萨芬的678"}
  • pk = id

测试用例 3: 包含特殊字符的参数

https://api.example.com/search?q=hello%20world&tags=tag1,tag2&date=2024-01-01
预期解析结果：
  • q = hello world
  • tags = tag1,tag2
  • date = 2024-01-01

测试用例 4: 空值参数

https://api.example.com/data?param1=¶m2=value2¶m3
预期解析结果：
  • param1 = (空值)
  • param2 = value2
  • param3 = (空值)

测试步骤：

  1. 打开Vue API Tester应用
  2. 将上述测试URL复制粘贴到URL输入框中
  3. 点击输入框外部或按Tab键失焦
  4. 检查Query参数列表是否正确显示解析出的参数
  5. 在Query参数列表中修改、删除或禁用参数
  6. 检查URL输入框是否实时更新显示

预期行为：

  • ✅ URL中的查询参数能正确解析到Query参数列表
  • ✅ 中文和特殊字符能正确处理
  • ✅ Query参数列表的变化能实时反映到URL中
  • ✅ 禁用的参数不会出现在URL中
  • ✅ 空值参数能正确处理
